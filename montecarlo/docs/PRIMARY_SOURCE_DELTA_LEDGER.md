# Baseline Primaire - Delta Ledger

> Date de creation : 2026-03-23
> Statut : actif
> Usage : registre append-only des ajouts documentaires sources, pour eviter de relire toute la baseline a chaque intervention

---

## But

Ce fichier sert a enregistrer uniquement les **nouveaux apports documentaires importants** :

- nouvelle source retenue ;
- nouvelle exigence projet derivee d'une source ;
- correction d'une exigence devenue obsolete ;
- preuve que le code a ete aligne ;
- preuve que la validation a ete executee.

La baseline longue reste le socle.
Ce ledger sert a **signaler ce qui a change depuis la baseline**.

---

## Regles

1. Une entree = un changement documentaire important.
2. Toute entree doit avoir un identifiant stable.
3. Toute exigence doit referencer au moins une source (`SRC-*`) ou une regle interne explicite.
4. Toute entree doit distinguer :
   - `proposed`
   - `accepted`
   - `implemented`
   - `validated`
   - `superseded`
   - `rejected`
5. On n'efface pas les anciennes entrees ; on les supersede.

---

## Format obligatoire

| Champ | Description |
|---|---|
| delta_id | identifiant stable (`DELTA-YYYYMMDD-XX`) |
| domain | backend / security / ml / risk / backtest / observability / docs |
| source_ids | liste d'identifiants `SRC-*` ou `INT-*` |
| requirement_ids | liste d'identifiants `REQ-*` concernes |
| change_type | source_addition / requirement_addition / correction / implementation / validation |
| status | proposed / accepted / implemented / validated / superseded / rejected |
| applies_to | chemins ou modules concernes |
| summary | resume court, lisible en moins de 20 secondes |
| proof_code | fichiers ou PR internes qui prouvent l'alignement |
| proof_validation | test, audit, import ou verification associee |
| supersedes | entree remplacee le cas echeant |
| notes | nuance importante |
